## What is BankScrap?

BankScrap is a Ruby gem that allows you to extract account balance and transactions from banks. You can use it either as a command line tool or as a library in your Ruby applications.

Most banks don't offer public APIs, and the only way to access your data (balance and transactions) is through their websites. BankScrap solves this problem by automating access to your banking data using two approaches:

- **Web scraping** on the bank's website
- **Reverse engineering** the bank's mobile app or single page web app APIs

## Supported Banks

BankScrap supports the following banks through dedicated adapter gems:

- **BBVA** (personal accounts) - `bankscrap-bbva`
- **BBVA Net Cash** (business accounts) - `bankscrap-bbva-net-cash`
- **ING Direct** - `bankscrap-ing`
- **Arquia Banca** - `bankscrap-arquia`
- **Openbank** - `bankscrap-openbank`
- **Sabadell** - `bankscrap-sabadell`
- **Ibercaja** - `bankscrap-ibercaja`
- **Santander** (WIP) - `bankscrap-santander`
- **Shinsei Bank** - `bankscrap-shinsei`

<Note>
  Interested in adding support for another bank? Open a new issue on GitHub and the community will help you get started.
</Note>

## Why BankScrap?

Developers shouldn't waste time on manual tasks that can be automated. Having to perform 20 clicks on a bank website just to check your balance or download transactions is frustrating and time-consuming.

BankScrap automates this process, allowing you to:

- Check your balance programmatically
- Fetch transactions for any date range
- Export data in standard formats (CSV, JSON)
- Integrate banking data into your applications
- Build financial automation tools

## Requirements

<Warning>
  Some banks require a JavaScript runtime to work properly. If you encounter an error like "Could not find JavaScript runtime", install Node.js on your system.
</Warning>

BankScrap requires:
- Ruby ~> 2.1 or higher
- A JavaScript runtime (Node.js recommended) for some banks
